# Bounded Context Canvas: Chat

## 1. Имя
**Chat** — ограниченный контекст обработки обращений клиентов в чате поддержки.

## 2. Назначение
Управление жизненным циклом сессии чата: от момента обращения абонента до закрытия сессии. Обеспечивает маршрутизацию между ботом и оператором, сбор обратной связи.

## 3. Стратегическая классификация

- [ ] Supporting
- [ ] Generic

## 4. Роль домена
- [x] Execution (исполняет бизнес-операции)
- [ ] Gateway
- [ ] Analysis

## 5. Входящая коммуникация (Inbound)

| Кто вызывает | Что вызывает | Формат |
|-------------|-------------|--------|
| UI абонента | SubscriberRequestQuestion | HTTP POST |
| UI абонента | SubscriberGaveFeedback | HTTP POST |
| UI абонента | SubscriberDownloadMedia | HTTP POST |
| Система маршрутизации | OperatorDequeueRequest | HTTP POST |
| UI оператора | OperatorRepliedToMessage | HTTP POST |
| UI оператора | OperatorClarifiedQuestion | HTTP POST |
| UI оператора | OperatorDownloadMedia | HTTP POST |
| Бот-сервис | BotRepliedToUser | Message Bus |
| Триггер-сервис | SessionEndingByTrigger | Message Bus |

## 6. Исходящая коммуникация (Outbound)

| Кому отправляет | Что отправляет | Формат |
|----------------|---------------|--------|
| Бот-сервис | ChatSessionCreated (новый вопрос для бота) | Event Bus |
| Система маршрутизации | ChatReadyForOperator (бот не справился) | Event Bus |
| Аналитика | SessionClosed (завершённая сессия) | Event Bus |
| Уведомления | FeedbackReceived | Event Bus |

## 7. Ubiquitous Language

- **Абонент (Subscriber)** — клиент, обратившийся в чат
- **Оператор (Operator)** — сотрудник поддержки
- **Бот (Bot)** — автоматический ответчик
- **Сессия (Session)** — один цикл обращения от вопроса до закрытия
- **Обращение (Request)** — вопрос абонента
- **Обратная связь (Feedback)** — оценка работы оператора/бота
- **Дежурная очередь (Dequeue)** — взятие обращения оператором

## 8. Бизнес-решения

- Сессия создаётся только абонентом (не оператором, не ботом)
- Оператор может взять обращение только один раз (нельзя перехватить)
- Оператор может отвечать и уточнять только после взятия обращения
- Абонент может оценить работу только в активной сессии
- Сессия закрывается триггером (таймаут, явное завершение)

## 9. Предположения

- Один абонент ведёт одну сессию одновременно
- Бот отвечает асинхронно через шину событий
- Медиа-контент хранится во внешнем хранилище, Chat хранит только ContentId

## 10. Зависимости / интеграции

| Контекст | Тип интеграции |
|----------|----------------|
| Bot | ClientSuppliers (Chat — потребитель ответов бота) |
| Routing | Partners (двусторонний обмен: маршрутизация ↔ статус) |
| Notification | OpenHostService (Chat публикует события, Notification подписан) |
| Analytics | PublishedLanguage (Chat публикует события в стандартном формате) |
| MediaStorage | SeparateWays (Chat хранит только ContentId) |
